OVERVIEW                                      *nvim-workspace-overview*

nvim-workspace provides workspace-aware navigation, search, explorer, and
session policy for Neovim. It keeps path/root policy in one place, then exposes
Telescope-backed file and grep pickers, optional Neo-tree helpers, and optional
persistence.nvim session helpers built on that policy.

The default behavior is intentionally broad: file and grep pickers search from
HOME unless a caller passes an explicit root. Users can switch root from inside
the picker without changing Neovim's process cwd.

==============================================================================
SETUP                                             *nvim-workspace-setup*
>
  require("nvim_workspace").setup({
    large_root_detector = function(root, opts)
      return false
    end,
  })
<
Options:

                                                        *nvim-workspace-large_root_detector*
large_root_detector ~
  Optional function called as `fn(root, opts)`. Return true when broad scans
  should be suppressed for that root. When unset, nvim-workspace falls back to
  a global `in_large_repo` function for compatibility with existing configs.

==============================================================================
COMMANDS                                       *nvim-workspace-commands*

                                                        *:WorkspaceFiles*
:WorkspaceFiles [dir] ~
  Open the workspace file picker. When [dir] is supplied, use it as the search
  root.

                                                        *:WorkspaceGrep*
:WorkspaceGrep [dir] ~
  Open the workspace grep picker. When [dir] is supplied, use it as the search
  root.

API                                                 *nvim-workspace-api*

Use `require("nvim_workspace")` for picker and root integrations. Optional
public integration modules are `nvim_workspace.neo_tree` and
`nvim_workspace.session`. Modules below `nvim_workspace.core` and
`nvim_workspace.picker` are internal implementation modules; they are not part
of the public API even though the test suite imports them directly.

                                                        *nvim_workspace.setup()*
setup({opts}) ~
  Configure plugin behavior.

                                                        *nvim_workspace.files()*
files({opts}) ~
  Open the file picker.

                                                        *nvim_workspace.grep()*
grep({opts}) ~
  Open the grep picker.

                                                        *nvim_workspace.register_file_source()*
register_file_source({source}, {opts}) ~
  Register an additional file source. The source receives
  `(prompt, done, root, ctx)`. Call `done(paths, { partial = true })` to stream
  partial results, then call `done(paths)` once to finish. Return a function or
  handle with `cancel()`/`kill()` when work can be cancelled.

                                                        *nvim_workspace.register_grep_source()*
register_grep_source({source}, {opts}) ~
  Register an additional grep source. Sources receive the same callback shape
  as file sources and should return ripgrep `--vimgrep` style lines.

                                                        *nvim_workspace.default_root()*
default_root() ~
  Return the broad default search root.

                                                        *nvim_workspace.current_file_dir()*
current_file_dir() ~
  Return the current real file buffer's directory, or nil for special buffers.

                                                        *nvim_workspace.current_buffer_dir()*
current_buffer_dir() ~
  Return the current file directory, falling back to Neovim's cwd for UI
  contexts that need a usable default.

                                                        *nvim_workspace.repo_root()*
repo_root({start}) ~
  Return the detected workspace root for {start}, or a normalized fallback.

==============================================================================
NEO-TREE                                       *nvim-workspace-neo-tree*

Use `require("nvim_workspace.neo_tree")` for optional Neo-tree integration.

filesystem_policy({root}) ~
  Return per-root Neo-tree git-status and filtering policy.

filesystem_mappings() ~
  Return mappings that mirror picker root controls: CTRL-R for HOME, CTRL-W
  for workspace root, and CTRL-D to prompt for a directory.

setup_manager_patch() ~
  Patch Neo-tree manager boundaries so reveal paths preserve visible HOME
  symlinks and per-root policy applies to navigation entry points.

open({root}, {opts}) ~
  Open Neo-tree with nvim-workspace root and reveal policy.

==============================================================================
SESSIONS                                      *nvim-workspace-sessions*

Use `require("nvim_workspace.session")` for optional persistence.nvim
integration.

load() ~
  Load the current persistence session when Neovim started without file
  arguments, then repair stale buffers and filetype state.

save() ~
  Save the current persistence session and file-root aliases.

save_current({persistence_plugin}) ~
  Save through an injected persistence-like object. This is mostly useful for
  tests or advanced host integrations.
